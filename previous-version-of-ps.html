The Journey of Decentralized DNS

An analysis of prior art, revealing the need for a truly fault-tolerant solution.

The Starting Point: Traditional DNS

The journey begins with the current DNS system. It's hierarchical and managed by central authorities like ICANN.

Key Drawback: Centralization

This model creates single points of failure, making it vulnerable to outages, security attacks, and censorship.

First Attempt: Namecoin

Namecoin was the first to combine DNS with a blockchain, creating a new, independent chain with a Proof-of-Work (PoW) consensus mechanism to prevent censorship.

Key Drawback: Poor Security

Its PoW network had insufficient computing power, making it highly vulnerable to a 51% attack, where a single entity could rewrite the DNS records.

An Improvement: Blockstack

Blockstack learned from Namecoin's security flaws and anchored its naming system to the powerful Bitcoin blockchain, inheriting its immense security.

Key Drawback: Low Performance

Its performance was limited by Bitcoin's 10-minute average block time and low transaction throughput, making it too slow and inefficient for a dynamic DNS service.

A Step Forward: ConsortiumDNS

The "ConsortiumDNS" paper proposed using a permissioned blockchain, which is better suited for organizations. It introduced a clever three-layer architecture to solve storage issues.

Three-Layer Architecture:

1. Storage Layer: Holds actual DNS data off-chain.
2. Blockchain Layer: Stores only lightweight transaction data and pointers.
3. Consensus Layer: The P2P network of nodes that maintains the system.

Key Drawback: Undefined & Inefficient Consensus

The paper explicitly states its system has low throughput and that a more efficient, fault-tolerant consensus mechanism is needed for future work. It identifies the problem but doesn't solve it.

The Next Evolution: Our Project's Solution

Our project builds directly on this prior art, addressing the critical gap left by ConsortiumDNS.

Our Innovation: A Complete, Fault-Tolerant Protocol

We propose a specific, efficient, and resilient consensus algorithm—**Round-Robin Proof of Authority with a Timeout**—to solve the exact problem that previous systems identified but left unresolved.
