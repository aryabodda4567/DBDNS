Project Overview

This project involves the design and implementation of a secure, resilient, and decentralized Domain Name System (DNS) built upon a purpose-built permissioned blockchain. The proposed solution is a self-contained ecosystem for private or consortium networks where trust, data integrity, and high availability are paramount. We will develop a blockchain from scratch in Java, featuring a novel and highly efficient consensus mechanism: **Round-Robin Proof of Authority (PoA) with a Timeout**.

Abstract

The system leverages a custom Proof of Authority (PoA) consensus algorithm, implemented as a deterministic Round-Robin Leader Schedule with a fault-tolerant Timeout mechanism. This approach guarantees high availability and eliminates the computational overhead of traditional systems. The architecture consists of authorized Leader Nodes for block validation and Normal Nodes for query processing. All DNS records are stored immutably on-chain, ensuring data integrity and resistance to censorship. The result is a secure, resilient, and efficient DNS framework suitable for private and consortium networks.

Phase 3: Consensus Mechanism

Goal: Implement the Round-Robin Proof of Authority logic, which is the heart of the system's efficiency and fault tolerance.

Consensus Loop: If Leader

  • Gather pending transactions.
  • Create a new Block.
  • Sign the block.
  • Broadcast block to peers.
  • Reset creation timer.

Consensus Loop: If Not Leader

  • Start a timeout timer (e.g., 15s).
  • Listen for a block from the leader.
  • If block received: add to chain, move to next leader.
  • If timeout: assume leader failed, move to next leader.
Phase 4: Networking and API

Goal: Enable nodes to communicate with each other (P2P) and with end-users (REST API).

P2P Communication

Use Java Sockets for a listener thread to receive broadcasted blocks and transactions from peers.

REST API Endpoints

  • POST /transactions/new
  • GET /dns/:domain
  • GET /chain
  • GET /status
Phase 5: Integration and Final Application

Goal: Tie all modules together into a single runnable application and define the testing strategy.

Testing Strategy

Run multiple instances on different ports to test the full lifecycle: registration, broadcast, block creation, peer synchronization, and leader failure/timeout.
